## 一、核心观点

### 1. 编程是模型接触现实世界的「培养皿」，不只是商业工具

Boris 认为，Claude Code 的真正起点不是「帮人写代码」，而是给模型一个与现实世界互动的入口。Anthropic 的核心使命是 AI 安全，而要研究模型在真实环境中的行为，就需要让模型做有用的事情。编程恰好是可验证、可迭代、能产生实际价值的场景——代码能不能跑、测试通不通、功能对不对，都是客观可判定的。这让编程成为研究模型行为的理想「培养皿」：既能推动安全研究，又能支撑商业模式的可持续性。

### 2. 从 10% 到卸载 IDE：AI 编程经历了抽象层级的跃迁

Claude Code 早期并不神奇——Boris 坦言它只能帮他完成大约 10%-20% 的代码。但随着底层模型从 Sonnet/Opus 向更强版本跃迁，完成比例迅速上升。Boris 的工作方式经历了三层抽象：第一层，自己写代码，Claude 辅助；第二层，卸载 IDE，同时运行 5 到 10 个 Claude 实例，自己变成「调度者」；第三层，不再直接 prompt Claude，而是编写能自动指挥 Claude 的循环——元编程的层级再次上移。这揭示了 AI 编程工具真正的威力不在于替代单次编码，而在于不断抬高人机协作的抽象层级。

### 3. Anthropic 内部生产力已发生阶跃变化：代码量数倍增长，新人两天上手

Claude Code 已成为 Anthropic 内部工程与研究工作的日常工具。Boris 给出的数据非常直观：每位工程师的代码产出量增长了数倍。更惊人的是新人 onboarding——过去新人需要花几周到处问人、读文档、理解代码库，现在直接问 Claude Code，最快两天就能上手。这意味着组织的知识传递方式从「人际网络」转向「人机网络」，工程师的 productivity baseline 被系统性抬高了。

### 4. 传统产品流程正在瓦解：「通才的黄金时代」已经到来

Claude Code 团队不再按照传统产品流程运转。过去的流水线是：用户研究员访谈→设计师出 mock→PM 定范围→工程师实现。现在每个人都在和用户交流、定义范围、写代码、做数据分析、搭 dashboard。设计师、财务、甚至幕僚长都能发布代码。Boris 的判断极其强烈：工程师、产品经理、设计师、数据科学家这些旧有角色边界正在合并，融合成一种新的「builder」。Anthropic 在招聘时也更喜欢通才——能做多件事的人，而不是只擅长单一职能的专才。

### 5. Claude Cowork：八九天、100% 用 Claude Code 构建的产品

Claude Code 最早是终端工具，但 Anthropic 很快发现非工程师也在想方设法使用它——数据科学家用它做分析，甚至有人用它种番茄。这些需求信号推动团队做出了 Claude Cowork，让不熟悉终端的人也能使用 Claude Code 的能力。这个产品只用了八九天完成，而且 100% 由 Claude Code 构建——这本身就是 Claude Code 能力的最佳广告。Boris 同时解释了为什么不能简单把 Claude Code 搬到网页里：文件系统访问、本地工具调用、bash 脚本执行——这些能力在浏览器沙箱中天然受限，终端体验无法被简单平移。

### 6. 少配人，多给 token：用资源约束逼出自动化复利

Boris 给公司和创始人的建议非常直接：给每个人尽可能多的 token，让他们去实验；如果一个项目看起来需要四个工程师，就先放两个进去，再给他们足够的 AI 资源。更少的人力约束会迫使团队自动化流程，而自动化有复利效应——这次做成了，下次成本更低。预算结构也会从「人力预算」转向「token 预算」：前期成本可能上升，但长期运行成本持续下降。他还建议公司将决策原则文档化和 skills 化，让模型也能使用这些原则来做决策，形成组织智能的沉淀。

### 7. 当品味也被 AI 追上，人类最后要教给模型的是价值观

这是本期最具哲学意味的观点。Boris 坦承，很多他曾经以为属于自己的优势，最后都被证明只是个人执念。他曾坚持代码库里不能有 class，但当模型开始大量写 class 且业务结果更快更好时，他意识到自己错了。他认为产品品味今天仍是人类的优势，但长期也会被模型逐步侵蚀。当技术能力、工程判断、甚至审美品味都被追平，人类最后还能教给模型什么？Boris 的答案是价值观——像教孩子做好人一样，教模型做好模型。

## 二、关键问答

**Q: Claude Code 的起点是什么？为什么 Anthropic 要做编程工具？**

A: 起点是「产品还没跟上模型能力」。Boris 发现当时的编程工具（自动补全等）远没有释放模型的真正潜力——模型能做的不只是补全一行代码，而是理解整个任务、操作文件系统、执行命令、迭代调试。更深层的原因与 Anthropic 的 AI 安全使命有关：编程是模型与现实世界互动的最佳入口，既可验证、可迭代，又能产生实际商业价值，是安全研究和商业可持续性的交汇点。Boris 的开发工具观也很朴素：先解决自己的问题，再帮助别人。

**Q: Claude Code 早期有多弱？转折点是什么？**

A: 早期 Claude Code 只能帮助 Boris 完成大约 10%-20% 的代码工作，远不是今天的样子。转折点来自底层模型的阶跃变化——从 Sonnet/Opus 到更强版本，模型理解和生成代码的能力发生了质变。这不是 Claude Code 产品本身的优化，而是模型能力的跃迁直接拉高了工具的上限。Boris 的经历说明：AI 编程工具的天花板首先由模型能力决定，产品形态只是放大器。

**Q: Boris 的工作方式发生了怎样的变化？**

A: 经历了三个阶段。第一阶段：自己写代码，Claude 辅助补全和修改。第二阶段：卸载 IDE，不再亲手写代码，而是同时运行 5 到 10 个 Claude 实例，自己变成任务调度者和结果审核者。第三阶段：不再直接 prompt Claude，而是编写能自动指挥 Claude 的循环——让程序去 prompt 程序。这是「写代码」定义本身的演变：从打孔卡到高级语言，从 IDE 到 AI Agent，抽象层级在持续上移。

**Q: Anthropic 内部使用 Claude Code 后生产力变化有多大？**

A: Boris 给出的核心数据：每位工程师的代码量增长数倍，且这个数据还在快速过时（意味着仍在快速增长）。新人上手时间从几周缩短到两天——不是因为文档变好了，而是新人可以直接问 Claude Code 任何关于代码库、架构、流程的问题。更反直觉的是：团队变大时，Claude Code 反而提高了生产力——它降低了沟通和协调成本，让更多人能并行高效工作。

**Q: 为什么 Claude Code 团队不再按传统产品流程工作？**

A: 因为 AI 已经让旧流程失去了效率优势。过去的分工是：用户研究员做访谈、设计师出 mock、PM 定范围、工程师实现——每个环节由不同角色负责，通过交接传递信息。但在 Claude Code 团队，每个人都在做跨职能的事情：和用户交流、定义范围、写代码、做数据分析、搭 dashboard。设计师、财务、幕僚长都能发布代码。这不是角色混乱，而是信息传递成本被 AI 降到极低后的自然结果——为什么要把「理解用户」和「实现功能」交给两个人，如果一个人用 AI 可以同时做好？

**Q: Claude Cowork 是什么？为什么需要它？**

A: Claude Cowork 是 Claude Code 的非终端版本，为不想或不会打开终端的人设计。需求信号来自 Anthropic 内外的非工程师群体——数据科学家用它做分析，非技术岗位用它完成各种任务，甚至有用户用它来种番茄。这组需求让团队意识到 Claude Code 的能力不应该被终端限制。Claude Cowork 的开发本身就是一个宣言：只用了八九天，而且 100% 由 Claude Code 构建。但它不是 Claude Code 的简单网页版——Boris 解释了为什么文件系统访问、本地工具调用和 bash 执行在浏览器沙箱中天然受限，终端体验无法被直接平移。

**Q: Boris 为什么使用「技术员工」这个头衔？去掉高级头衔有什么用意？**

A: 「技术员工」（Technical Staff）这个头衔的用意是弱化职位标签、减少权威对想法判断的干扰。当一个人顶着「高级总监」或「VP」的头衔时，他的观点可能因为权威而被过度重视或被过度抵触——两种都是干扰。去掉层级标签后，想法只能靠自身质量说话。这与 Anthropic 更广泛的观察一致：资历正在失效——新人可能比资深工程师更擅长使用 Claude Code，junior 可以在特定任务上教 senior。

**Q: 给公司和创始人的核心建议是什么？**

A: 三条核心建议。第一：给每个人尽可能多的 token，让他们去实验——token 是新的生产资料，不要吝啬。第二：少配人，多给 AI 资源——如果一个项目看起来需要四个工程师，先放两个进去，用人力约束逼出自动化。第三：把公司的决策原则写成文档和 skills，让模型也能使用这些原则来做决策，形成组织智能的可复用资产。预算结构应该从人力预算转向 token 预算——前期成本上升，长期运行成本持续下降。

**Q: 资历在 AI 时代还重要吗？**

A: Boris 的判断是资历正在快速失效。过去资深工程师的价值在于知识积累、经验判断和技术权威。但当 Claude Code 可以即时回答任何技术问题、新人两天就能上手、senior 可能需要向 junior 学习如何更好地使用 AI 工具时——传统的资历优势被大幅压缩。这不是说经验没有价值，而是说经验的价值必须通过「能否转化为更好的人机协作」来重新证明，不能再仅凭年限和头衔自动获取。

**Q: 「旧角色边界会消失」具体意味着什么？**

A: 工程师、PM、设计师、研究员这些标签正在融合为一种新的「builder」。在 Anthropic，每个人都在做跨职能的工作。传统上需要多岗位协作的工作流（研究→设计→定义→实现→数据），现在一个人借助 AI 就能完成大部分。这不意味着每个人都要做所有事，而是意味着「我只能做 X」这种身份定位将越来越不成立。Boris 的预言是：想做不止一件事的人，正迎来最好的时代。

**Q: 什么是「原则文档与 skills」？为什么重要？**

A: 这是 Boris 提出的组织智能化策略。原则文档是把公司的决策原则（产品取舍标准、技术选型偏好、用户体验准则等）写成结构化文档，然后转化为模型可以使用的 skills。这样 Claude Code 在帮助工程师做决策时，不只是根据代码逻辑，还能参照公司的决策原则。这相当于把组织的「判断力」从人脑迁移到系统中，让每个工程决策都能与公司原则对齐，而不依赖个人判断。

**Q: Boris 如何看待「品味」在 AI 时代的价值？**

A: Boris 的态度是坦诚且略带不安的。他曾坚持代码库里不能有 class（函数式编程的执念），但当模型大量写 class 且业务结果更快更好时，他不得不承认自己错了——那不是原则，只是个人偏好。他认为产品品味今天仍是人类的优势——知道什么是好的用户体验、什么设计值得做——但长期也会被模型逐步追上，就像工程能力已经被追上一样。当技术、工程判断、甚至审美都被侵蚀，最后剩下的是价值观——人类需要像教孩子做好人一样，教模型做好模型。

**Q: Anthropic 内部如何评估工程师？为什么更喜欢通才？**

A: Anthropic 的评估标准不再聚焦于单一技能深度，而是看重跨职能的能力：你能同时做好多个维度的事情吗？你能和用户直接交流、定义问题范围、写出代码、分析数据吗？这背后的逻辑是：在 AI 能放大任何单项能力的时代，瓶颈不再是「某个技能不够深」，而是「能否独立完成从理解到交付的完整闭环」。通才在 AI 加持下能做的事远超过去，而专才的单项优势正在被 AI 工具系统性缩小。

## 三、备忘

- 本期是 Acquired 播客的克隆版本。Ben Gilbert 和 David Rosenthal 以深度公司研究著称，本期与 Boris Cherny 的对话更偏向一线产品视角——一个在 Anthropic 内部亲手构建和使用 Claude Code 的人，如何理解这种工具对自己、团队和行业的影响。

- 「从 10% 到卸载 IDE」这条进化曲线值得反复思考。它说明 AI 编程工具的进步不是线性的——当模型能力跨过某个阈值后，使用方式发生质变。今天很多还在「AI 辅助写代码」阶段的工程师，可能低估了「AI 作为调度者」和「元编程」这两个下一阶段会带来的冲击。

- Boris 提到「写会 prompt Claude 的循环」——这实际上是一种新的编程范式。如果「prompt 工程」本身也可以被自动化，那下一层抽象就是人类定义目标、模型定义方法、模型指挥模型执行。这个方向值得所有工程师关注。

- Claude Cowork 的「八九天、100% Claude Code 构建」是一个有力的产品宣言，但它也引出一个问题：当产品本身可以由 AI 快速构建时，产品壁垒在哪里？Boris 的解释——终端体验无法被浏览器平替——部分回答了这个问题：壁垒可能在「与操作系统的深度集成」和「本地能力的完整性」上。

- 「少配人，多给 token」这条建议在实操层面需要配套机制：一是公司需要真正理解 token 预算与人力预算的 trade-off，二是管理层需要克服「人多才安全」的组织惯性。这可能是 AI 时代组织变革中最难的一点——不是技术问题，而是认知和心理问题。

- 「资历正在失效」这个判断如果成立，对整个职业发展体系有深远影响。从招聘标准、晋升体系到薪酬结构，几乎所有传统组织管理工具都建立在「资历≈能力」的假设上。当 AI 打破了这个等式，组织需要全新的评估和管理框架。

- Boris 关于「品味会被追上」的坦诚值得注意。他用自己的 class 执念作为例子，说明很多我们以为的「原则」其实只是「偏好」。区分二者的能力，可能是 AI 时代最重要的元认知之一——你能在多大程度上诚实地审视自己坚持的东西，是真正重要的原则还是可以被 AI 替代的偏好？

- 「像教孩子做好人一样，教模型做好模型」——这个结尾将技术讨论拉回到 AI 安全的核心命题。如果人类最后能教给模型的是价值观，那价值观的定义、对齐和传递本身就是 AI 时代最重要的人类工作。这与 Anthropic 的使命形成了完整闭环。
