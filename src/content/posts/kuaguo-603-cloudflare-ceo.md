## 一、核心观点

### 1. 2026年机器人流量首次超过人类：互联网的「图灵拐点」已经到来

Cloudflare 平台上监测到一个里程碑式数据：2026年上半年，机器人流量首次超过了人类流量。Matthew 指出，互联网上一直有机器人在活动，但比例长期稳定在20%左右；大约两年前开始真正激增，他在2025年秋天曾预测自动化流量会在2027年底超过人类，今年3月把时间点提前到2027年上半年，结果几周前团队告诉他——2026年上半年就已经超过了。更惊人的是五年后的预测：机器人流量可能是人类的一千倍。他纠正了一个常见的语义误区——推动增长的并非传统爬虫，而是 AI Agent。一个人类挑选数码相机可能访问五个网站，一个 Agent 则会遍历五千个。从基础设施角度看，这意味着互联网上将产生巨大的额外需求，五年后的互联网流量可能是今天的一千倍，且这个数字很可能被低估，因为它正以指数级速度增长。

### 2. 广告商业模式的终结：机器人不会点广告，互联网需要全新的价值交换机制

过去28年，互联网的底层商业模式基本没变——主要靠广告。但 Matthew 指出一个简单的致命事实：机器人不会点广告。在一个 Agent 主导的网络世界里，广告展示失去了它的经济意义。更根本的问题是，运营一个流量暴增一千倍的互联网，成本会高得多，总得有人为此买单。他提出了几个可能的方向：内容拥有者需要控制谁能访问自己的内容、微支付机制的复兴（HTTP 状态码 402 本来就是"需要付款"）、内容与 AI 公司之间更直接的授权协议。他透露全球最大的出版商正在和 AI 公司签下比过去好得多的协议，并认为互联网过去28年的商业模式其实并不健康——"流量从来都是价值的糟糕替代品"，大量媒体靠制造愤怒和分裂来驱动点击，如今世界的很多分裂都源于这个以流量为基础的底层商业模式。

### 3. Cloudflare 的真正故事：不断给自己制造问题，然后被迫去解决

Matthew 把 Cloudflare 的演化总结为一个不断自我制造问题的过程。15年前注册时，想法很简单——"把防火墙放到云里"，在网站前面放一个门卫。最初的理念是以易于采用的打包方式提供性能和安全性，创建免费版本让每个人都能用上。但在服务客户的过程中，发现预算越少安全问题越少、预算越多安全问题反而越多，只有两类组织打破这个规律——黑客小孩和人权组织。于是被迫去做域名注册、DDoS 防御、DNS 基础设施，又不得不搭建自己的开发者平台。今天的 Cloudflare 跟刚起步时完全不一样：从全球五个城市的服务器起步，到现在遍布超过350个城市、超过一千个数据中心。他直言没有写过什么宏大战略备忘录——做开发者平台是因为自己需要，服务巴基斯坦是因为巴基斯坦电信主动发了邮件，而巴基斯坦国家板球队是 Cloudflare 的免费客户。

### 4. 离科技中心越远，Cloudflare 提供的价值越大——互联网基础设施的「民主化」效应

Matthew 分享了一个早期 VC 的经典问题："你们怎么扩展到湾区以外？"他的回答是——在突尼斯这些地方 Cloudflare 非常火，但在湾区反而没什么用户。"离科技中心越远，我们提供的价值就越大。"如果你去刚果，当地排名前一百的网站里有50%都在用 Cloudflare。这个现象背后的逻辑是：科技中心的公司天然拥有更多技术资源来自己解决问题，而远离中心的人和组织的互联网体验完全依赖外部基础设施。Cloudflare 无意中成为了一种互联网民主化力量——巴基斯坦电信这样"极其难打交道"的组织主动找来合作，土耳其 escort 网站的流量模式最终帮助识别并追踪了一个被伊朗军事体系吸纳的黑客。Matthew 把这称为"服务所有人的力量"——从互联网奇奇怪怪的角落学到的东西，最终保护了 Eurovision 歌唱比赛（观众比 Super Bowl 还多）和 JP Morgan。

### 5. 未来两年每周都会发生 Log4J 级别漏洞——AI 既是攻击者也是修复者

Matthew 对 AI 时代的安全前景做出了一个令人毛骨悚然的预判："未来两年，网上会接连发生非常可怕的事。接下来104周，每周都会看到一个 Log4J 级别的漏洞。"他说当前的 AI 模型找漏洞的能力强得惊人。但话锋一转，他的真正观点是：两年后软件质量会因此大幅提升。Cloudflare 已经在内部实践这个逻辑——构建了一个 Agent 来审查每一次代码发布和配置变更，用10年的事故数据训练。结果是在线时间、可靠性和性能在过去一年里提升了一个数量级。Matthew 特别强调了一个细节：AI 跟组织内部所有其他偏见都不相关——它不会累，不需要休息，不用睡觉，也不会被办公室政治影响判断。这种"不相关"恰恰是 AI 在安全审计领域最被低估的优势。

### 6. 裁员20%不是成本削减，而是组织架构的必然重构——「最善良的事就是尽快动手」

Matthew 坦言 Cloudflare 裁掉了超过20%的团队，并直言其他公司的 CEO 同行也都看到了同样的情况，但"很多领导者心里其实很怕，不想当第一个动手的人"。他的判断是：未来6到12个月，几乎每家公司都会经历类似的调整。他的核心理由不是成本，而是组织效率的根本性变化——管理幅度从6:1变成了12:1，AI 让每个管理者能带更多人。他拿出了极其慷慨的补偿方案，让被裁员工的股票继续归属，因为他相信"一旦你意识到这种改变必然会发生，你能为团队做的最善良的事，就是尽快动手——因为今天找工作比6到12个月后容易得多。最糟糕的情况是6到12个月后出现大血洗。"他还观察到一个前所未有的现象：大量资深经理人主动举手说想回去做个人贡献者（IC），这在以前几乎不可想象。

### 7. 人类历史上第一次拥有了关于所有人类知识的数学模型——但里面有很多洞需要填

Matthew 用一个出人意料的比喻来描述大语言模型对人类知识的掌握——"画面像一大块瑞士奶酪，奶酪很多，但里面也有很多洞。"AI 公司现在最想要的是声誉极高的内容来源去填补那些洞、贡献全新的知识。他由此提出了一个更大胆的设想：应该搞一个类似奥斯卡或诺贝尔奖的机制，专门奖励对人类知识贡献最大的人，因为一个精准填补知识空白的行为，其长期价值可能远超一次商业成功。他自己的实践是买下了家乡犹他州帕克城当地的报纸，并预测未来 AI 授权收入可能超过广告收入。他的反思是："过去28年的互联网几乎扼杀了本地、古怪、独特、有趣的信息"，而这些信息恰恰是 AI 时代最有价值的数据资产。

### 8. 流量从来都是价值的糟糕替代品——互联网需要一次「史蒂夫·乔布斯 99 美分」式的重新定价

Matthew 花了大量篇幅讨论互联网商业模式的根本缺陷。他飞到斯德哥尔摩见了 Spotify 创始人 Daniel Ek，从一个意想不到的角度切入——Spotify 把搜索效果不好的查询公开发布给音乐创作者，结果丹麦有一个人光靠给未被满足的搜索请求写歌，一年赚了4000万欧元。他把这个模式和 Steve Jobs 当年发布 iTunes、一首歌99美分的时刻并列——乔布斯"插下了一面旗帜"，告诉所有人数字内容是有价格的。如今互联网需要类似的旗帜：应该有一种微支付的方式。Cloudflare 网络每秒处理5亿个请求，哪怕只有1%到10%可以通过微支付变现，就是一个比 Visa（每秒不到10万笔交易）大两个数量级的支付网络。他提醒所有人：最初的 HTTP 响应码 402 就是"需要付款"——互联网的创始者们早就为这一天预留了接口，只是 Google 构建的广告模型在过去28年把它暂时掩盖了。

## 二、关键问答

**Q: 机器人流量超过人类这个趋势是什么时候开始的？**

A: 互联网上一直有机器人在活动，但比例长期稳定在20%。大约两年前机器人流量开始真正激增。Matthew 在2025年秋天曾预测自动化流量会在2027年底超过人类，2026年3月西南偏南大会上把时间点提前到2027年上半年，结果几周前团队告知——2026年上半年就已经超过了。他补充说，推动增长的真正力量不是传统爬虫，而是 AI Agent——一个 Agent 挑选数码相机会访问五千个网站，而人类只访问五个。

**Q: 五年后互联网会是什么样子？**

A: Matthew 的判断是五年后互联网流量可能是今天的一千倍，而且"如果让我打个赌，我会赌这个数字会更高，因为它正以指数级的速度增长。"这种增长来自 Agent 的大规模部署——每个知识工作者可能都有一个 Agent，其 CPU 占用量是全球年产量的40倍。但他同时指出，运营这样一个互联网的成本会高得多，而机器人不会点击广告，所以过去28年依赖广告的商业模式必须发生根本性改变。

**Q: 在 Agent 的世界里，品牌意味着什么？**

A: Matthew 认为这是一个开放且极其深刻的问题。当机器人有无限耐心去发掘所有可能合适的东西，传统的品牌溢价逻辑——消费者因为认知和信任而选择熟悉的品牌——就会被颠覆。Agent 不会被品牌广告影响，它只关心参数匹配和性价比。这意味着品牌的价值主张需要从"让人类在信息有限时做出安全选择"转变为"让 Agent 在信息无限时也能识别出不可替代的价值"。

**Q: Cloudflare 是怎么从"把防火墙放到云里"变成今天这样的？**

A: Matthew 说 Cloudflare 真正的故事是"我们一开始的想法很简单，然后在这个过程中给自己制造了太多问题，不得不去解决。在解决这些问题的过程中，又造出了一整套产品。"最初只是想在网站前放一个门卫，创建免费版本让每个人都能用。但在服务过程中发现两类最需要安全保护的客户——黑客小孩和人权组织——于是被迫去做域名注册、DDoS 防御、DNS 基础设施，后来又不得不搭建自己的开发者平台。他们没有写宏大战略备忘录，每一步都是被实际需求推动的。

**Q: 三个创始人是怎么分工的？16年没吵过架是真的吗？**

A: 三个创始人在创业初期不是朋友，但 Matthew 认为力量恰恰来自于他们如此不同。他引用 Michelle 的维恩图比喻——"每个人有刚刚好的重叠"。分工上，Lee 是技术天才，Michelle 是运营负责人（六西格玛黑带），Matthew"比较会讲故事"。Matthew 说 Michelle 和他之间在近17年里从来没吵过架。当创业者来取经时，问得最多的问题就是"你们当初怎么分工"。他透露 Lee 有时会想"为什么要来吃这顿饭"，而 Michelle 则用六西格玛方法论把一切流程化——三种截然不同的人格的互补性恰恰是公司长期稳定的秘诀。

**Q: 那个土耳其 escort 网站的故事到底是怎么回事？**

A: 这是 Cloudflare 历史上最传奇的故事之一。早期公司办公室挂了一个铃铛，每次有人注册 Cloudflare 铃铛就会响。有一天铃铛疯了一样响个不停——超过一千家土耳其色情网站注册了 Cloudflare。公司把网站翻译成了土耳其语，Matthew 联系了其中一位站长约 Skype 通话，从而了解到了这些网站正在遭受的一种特殊攻击模式，团队将其标记为 TE（Turkish Escort Attack）。一年后，Eurovision 歌唱比赛遭到攻击，系统识别出完全相同的 TE 攻击模式；又过了一年，JP Morgan CTO 打来紧急电话，团队再次认出同一个攻击者。最终溯源发现是一个伊朗学生，后来被吸纳进了伊朗军事体系。Matthew 的总结是：从互联网最奇奇怪怪的角落学到的东西，最终保护了全世界最重要的机构。

**Q: Cloudflare 在 AI 方面的布局是什么？什么时候开始的？**

A: Matthew 说 Cloudflare "一直都是一家 AI 公司"——真正的起点是 MIT 反垃圾邮件大会，那本质上就是一场机器学习会议。但现代篇章真正开始于2017年，当时 FedEx CEO 在宕机时打来电话说"你们掉线，我们有些飞机没法降落"，这迫使团队必须搞一套全新的软件开发和部署方式，催生了 Cloudflare Workers。2020年左右他们跟 NVIDIA 合作在边缘部署 GPU，几乎毫无反响；2024年发了一模一样的新闻稿、只改了日期，股价直接翻倍。如今几乎所有模型都可以通过 Cloudflare 直接访问且性能更好。他坦言边缘网络目前还不太适合做训练，但真正擅长的是推理场景。此外还推出了 AI Gateway——审计所有发往 AI 系统的 prompt 和响应、防止 prompt 注入、智能路由请求用简单模型处理简单任务来控制成本。

**Q: 为什么说 CPU 短缺危机的讨论还不够多？**

A: Matthew 用了一个惊人的数字来说明：如果每个知识工作者拥有一个 Agent，CPU 占用量将是全球年产量的40倍。他指出 Workers 围绕浏览器标签页的 isolate 概念构建——这是一种极度轻量级的计算单元，目的是在同样的硬件上跑更多工作负载。他甚至用 Rust 重写了 WordPress（内部代号 M-dash），因为如果流量真的暴增一千倍，"没人会再搭 WordPress 网站"。他担心整个行业对即将到来的计算资源需求爆炸缺乏足够的准备。

**Q: 93%的研发员工在用 AI 编程工具，但你们自己用起来其实挺谨慎的——为什么？**

A: Matthew 分享了一个生动的内部故事：Cloudflare 的 Kenton Varda（技术团队的关键人物）一开始说"这些 AI 玩意儿都是扯淡"，然后花了一个月试图证明它们有多烂，回来后却说"效率比以前高了一百倍"。Matthew 说真正的转折点大概在2025年11月——在此之前团队一直在谨慎验证，在此之后变成了全面拥抱。但他特别担心"处于职业生涯中期的人"——那些已经有一定经验但还远没到退休的人，可能会有一代被落下的人。工程团队天然是这些产品的用户，但其他部门（财务、法务、IR）则需要刻意设计和推动才能让 AI 真正融入工作流程。

**Q: Cloudflare OS 是什么？那个"神奇的 AI Agent 邮箱"又是怎么回事？**

A: Cloudflare OS 是 Matthew 推动的一个内部项目，目标是把 AI 能力带给工程之外的每个部门——让财务团队也能极其高效地完成工作。"神奇的 AI Agent 邮箱"是一个特别能说明问题的人机协作实验：搞一个邮箱地址，告诉所有人这是神奇的 AI Agent，但实际上后台是20人团队7天24小时轮班手动处理请求。这个实验的目的是先理解用户真正需要 Agent 做什么，再逐步用真正的 AI 替代人工。投资者关系团队的 Heather 是另一个典型案例——原本两周的流程被 AI 压缩到了三分钟。Matthew 说"每家公司都会经历这个过程"，并透露国际奥委会 CTO 已经主动联系说2034年犹他州冬奥会需要这个系统。

**Q: 裁员20%背后的真正逻辑是什么？对自己公司的人有什么建议？**

A: Matthew 不是 Dario（Amodei）那一派——他不认为 AI 会取代所有工作，但认为会发生巨大变化的地方是"既不负责构建也不负责销售的岗位"——中台、审计、合规、报告等。内部审计是一个典型例子：以前每季度只能覆盖6-10个风险领域，现在105个全部持续审计。管理幅度从6:1变成12:1，组织变得更扁平。他观察到大量资深经理人主动说想回去做个人贡献者。对个人的建议是：如果今年夏天招了1111名实习生，工程团队里92%的人都在用 AI 工具，这个数字应该达到100%——如果你还没有在用，你必须立刻开始。对那些可能被影响的人，他的建议是尽早转型，"不要等到6到12个月后的大血洗"。

**Q: Spotify 的 Daniel Ek 给了什么启示？那个一年赚4000万欧元的丹麦人是怎么回事？**

A: Matthew 飞到斯德哥尔摩见了 Spotify 创始人 Daniel Ek，讨论的核心问题是互联网的商业模式转型。Spotify 的做法出人意料——他们把搜索效果不好的查询公开发布给音乐创作者，让创作者知道"有人在搜索这种类型的歌但找不到"。结果丹麦有一个人光靠给未被满足的搜索请求写歌，一年赚了4000万欧元。Matthew 把这个模式和 iTunes 99美分的历史意义并列——Steve Jobs 用一首歌99美分"插下了一面旗帜"，告诉全世界数字内容不是免费的。他认为 AI 时代需要类似的旗帜来重新定价人类知识，而 Spotify 的"需求信号公开化"提供了一个让创作者直接对接到需求的新范式。

**Q: "瑞士奶酪"比喻是什么意思？为什么 Matthew 买下了家乡的报纸？**

A: Matthew 用"一大块瑞士奶酪"来比喻大语言模型对人类知识的覆盖——奶酪本身很多，但里面有很多洞（知识空白）。AI 公司现在最渴望的就是声誉极高的来源去填补那些洞、贡献全新的知识。他由此产生了一个想法：应该设立一个类似奥斯卡或诺贝尔奖的机制，专门奖励对人类知识贡献最大的人。这也是他买下家乡犹他州帕克城当地报纸的原因——他预测未来 AI 授权收入可能超过广告收入。他的深层反思是："过去28年的互联网几乎扼杀了本地、古怪、独特、有趣的信息"，而这些东西在 AI 时代恰恰是最有价值的——它们是瑞士奶酪上那些独特的纹理和味道，不是工业流水线上能被轻易复制的数据。

**Q: HTTP 状态码 402 是什么？为什么它很重要？**

A: HTTP 状态码 402 的官方含义是"需要付款"（Payment Required）。Matthew 指出，互联网的创始者们在设计协议的时候就为微支付预留了这个接口，但它从未被真正使用——因为 Google 构建的广告商业模式在过去28年主导了互联网，让一切看起来是"免费"的。现在 Cloudflare 的网络每秒处理5亿个请求，哪怕只有1%到10%可以通过微支付变现，就需要一个比 Visa（每秒不到10万笔交易）大两个数量级的支付系统。402 这个被搁置了近30年的状态码，可能在 AI Agent 时代迎来它本该拥有的功能。Matthew 认为，过去28年互联网的商业模式其实并不健康——"流量从来都是价值的糟糕替代品"，而 Agent 时代将迫使所有人重新回答一个基本问题：信息到底值多少钱？

**Q: 如果 Matthew 自己是一个普通知识工作者，他现在会做什么？**

A: 从整个对话中可以提炼出 Matthew 的隐含建议：第一，立刻开始使用 AI 工具——不是浅尝辄止，而是深度融入日常工作流程，像 Kenton Varda 那样从怀疑到"效率一百倍"；第二，关注自己工作中那些"既不负责构建也不负责销售"的部分是否在被自动化——这是目前变化最快的地带；第三，不要等到变化被迫发生再行动——"一旦你意识到改变必然发生，最善良的事就是尽快动手"这句话不仅是对 CEO 说的，也是对每个人说的；第四，留意那些被过去28年的互联网商业模式所扼杀的"本地、古怪、独特、有趣"的东西——它们可能成为 AI 时代最稀缺的价值来源。
