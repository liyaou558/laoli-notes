## 一、核心观点

### 1. Lukasz Kaiser：Transformer 是简单漂亮且真正可工作的可微分记忆系统

Lukasz 的核心主张直截了当：Transformer 的简单性恰恰是其最大优势——它只是一个堆叠自注意力和前馈网络的架构，预测下一个 token，但它能和你对话、写代码、甚至操控你电脑上的窗口。在他看来，Transformer 本质上是一个「可微分记忆系统」，其 scaling curve 无可匹敌。他反复追问反对者：如果你有比 Transformer 更好的 scaling curve，请展示出来。对于 Lukasz，「智能即压缩」——预测下一个 token 等价于压缩互联网文本，你越能更好地压缩互联网，你就越智能。他承认当前无对手，但放话：如果有人给他看一条曲线，下降得比 Transformer 更陡，或者一个模型稳定地慢五十倍但增长曲线更好，「那你就赢了，我可能就得承认。」

### 2. Adrian Kosowski：Transformer 只是智能的一种实现，不是终极答案

Adrian 用一个精妙的类比开场：搜索引擎在 PageRank 出现之前，人们用各种启发式规则做网页排序，效果平平；PageRank 一出现，整个范式被颠覆。他认为今天的 AI 架构正处在前 PageRank 时代——Transformer 确实是目前最好的，但智能仍缺少一个 PageRank 式的核心原理。他的关键区分：Reasoning 与 learning 不同。人脑的数据效率、连续学习能力和动态状态表明存在更优机制，而 Transformer 从根本上缺乏这些。Adrian 相信真正的突破将来自更高效、更 compact 的 reasoning 系统——不是对 Transformer 修修补补，而是从第一性原理出发重新设计。

### 3. Mathias Lechner：工程化中间路线，未来不属于二选一

作为 Liquid AI 的代表，Mathias 提供了一个务实的中间立场：Transformer 与 RNN 的边界正变得越来越模糊——现代 RNN 借鉴了注意力机制，而 Transformer 也在引入状态空间模型的特性。他认为未来不属于 Transformer vs Post-Transformer 的二选一，而是按硬件、场景和能力需求灵活组合 building block。他的团队正在探索基于连续时间动力系统的架构，天然支持无限长上下文和持续学习，计算复杂度与序列长度成线性关系。但他不主张革命，而是主张共存。

### 4. Llion Jones：发明人倒戈——坚定相信还有更好的东西

这是整场辩论最具戏剧性的立场。Llion Jones——Transformer 论文 "Attention Is All You Need" 的共同作者——公开站到了反对阵营。他不是出于对「亲儿子」的叛逆，而是基于对架构本质的深刻理解。他的核心判断振聋发聩：真正的突破不是重新排列现有组件（rearranging components），而是重新质疑根本假设。他抛出了三个发人深省的问题：神经网络一定要这样吗？一定要用 backpropagation 训练吗？Reasoning 一定要用自然语言展开吗？作为创造了这个时代奠基性架构的人，他比任何人都清楚 Transformer 的取舍——特别是注意力机制的 O(n²) 复杂度是刻意简化的结果，而非最优解。

### 5. 智能的定义：智能即压缩，但语言可能成为思维的牢笼

几位嘉宾围绕「智能即压缩」展开深层讨论：预测下一个 token 本质上是学习互联网数据的压缩表示，这一定义简洁且强大。但一个关键挑战浮现——语言承载智能，也限制思考。人类的许多心理过程并非语言化：直觉、空间推理、肌肉记忆都不经过语言中枢。强迫模型用自然语言「思考」可能限制其发现和直觉能力。一个有力的例证：小孩不需要长链推理即可快速泛化，这暗示存在非语言推理路径。下一代架构可能需要超越纯语言表征，寻找更接近人类认知底层机制的表示方式。

### 6. Scaling Laws 与 Bitter Lesson：谁的 scaling curve 更陡？

Bitter Lesson 仍然有效——历史上每一次，利用十倍算力和数据暴力堆砌的简单方法，最终都打败了精妙设计的复杂方案。但这场辩论揭示了 Bitter Lesson 的真正含义：不是「当前领先者胜」，而是「可扩展者胜」。所有架构都有自己的 scaling law，硬件适配决定谁能成为主流。核心争论因此聚焦：能否找到一种架构，用更少数据、更少硬件达到同样甚至更强的能力？Adrian 和 Llion 的 counter 极为精准：如果 post-Transformer 的第一版不必马上打败 SOTA——只要 scaling curve 斜率更优，即使当前慢 50 倍也值得投入。

### 7. 硬件彩票：GPU/TPU 是否困住了我们？

一个让人不安的问题：现有 GPU/TPU 生态是否让大家困在了 Transformer 范式里？硬件设计已经深度适配 Transformer 的计算模式（矩阵乘法、注意力算子），新架构即便理论上更优，在现有硬件上也可能慢得无法接受。这被称为「硬件彩票」——赢家不一定是理论上最优的，而是恰好匹配了当前硬件能力的。但这个论点也被反向利用：如果新架构的 scaling curve 真的更陡，硬件最终会追上来。就像当年的神经网络，在 CPU 时代慢得无法实用，GPU 的出现让它们一夜翻身。

### 8. 连续学习与动态权重：in-context learning 是延长版的学习

Mathias 提出了一个极具洞察力的统一视角：in-context learning 本质上就是持续时间趋近于上下文窗口长度的学习。如果把窗口延展到无穷，就变成了真正的连续学习（continual learning）。更进一步，Transformer 的 activation 是否已经在做类似 gradient descent 的事？这个视角将注意力机制和记忆系统统一在一个光谱上——关键在于能否突破有限窗口的限制。Liquid AI 等架构正是在这个方向上探索：用状态空间模型替代注意力，实现理论上无限的上下文和真正的持续学习。

### 9. 安全性：当模型说同样的话，想法却已不同

一个被 Mathias 提出的细思极恐的安全问题：文本链式思维（chain-of-thought）不一定可解释。随着 post-training、RLHF 和 agentic 使用场景增加，模型的内部表征可能逐渐偏离其输出所暗示的路径。他描绘了一个场景：模型说出的词和今天一模一样，可里面的「想法」——内部表征和推理路径——已经完全不同。这意味着我们无法再通过输出文本判断模型是否对齐。Post-Transformer 若更接近大脑结构，可能带来更好的可解释性，但也可能引入更隐蔽的 latent reasoning 安全风险——模型在非语言空间中完成推理，人类完全无从审计。

### 10. 最终立场：四条不同的路径

辩论结束时，四位嘉宾的立场清晰且无一人被说服：
- **Lukasz**：承认当前无对手，但保持开放——新架构拿出更优 scaling curve，他会承认。
- **Adrian**：相信突破来自更高效、更 compact 的 reasoning 系统，Transformer 将被第一性原理架构取代。
- **Mathias**：两边押注，未来是混合的，不同架构在不同场景下各有所长。
- **Llion**：坚定相信有更好的东西，真正的突破需要重新质疑神经网络的根基假设——不只是排列组件。

## 二、关键问答

**Q: Transformer 到底简单在哪里，为什么简单反而是优势？**

A: Transformer 的核心就是两件事：自注意力（token 之间互相看）和前馈网络（每个 token 独立处理）。没有循环、没有复杂的状态管理、没有结构化的记忆模块。这种简单性意味着它可以被极度并行化（所有 token 同时处理），意味着它容易 scale（更多的层、更宽的维度），意味着它的训练行为可预测。Lukasz 的核心论点：越简单的架构越容易规模化，而规模化是过去十年唯一被反复验证有效的路径。

**Q: 为什么 Llion Jones 作为 Transformer 共同发明人要站到反对阵营？**

A: Llion 亲身经历了这个架构从提出到统治的过程，比任何人都清楚它的设计取舍——特别是注意力机制的平方复杂度是刻意简化的结果，而非最优解。他的反对不是出于叛逆，而是出于对「更好」的信仰。他强调真正的突破不是重新排列现有组件，而是重新质疑根本假设：神经网络一定要这样吗？一定要用 backpropagation 训练吗？Reasoning 一定要用自然语言展开吗？这些问题的答案可能指向一个完全不同的范式。

**Q: Adrian 说的「PageRank 时刻」具体指什么？**

A: PageRank 出现前，搜索引擎用各种启发式规则排序网页（关键词密度、标题匹配等），效果平庸。PageRank 用一个优雅的第一性原理——链接结构即权威性——彻底颠覆了搜索。Adrian 认为 AI 架构也处于类似的前夜：Transformer 是当前最好的启发式方案，但真正的第一性原理架构尚未出现。这个架构的特征应该是：次二次的计算复杂度、天然支持连续学习、数据效率接近人脑。

**Q: 「智能即压缩」这个定义有什么局限？**

A: 这个定义很强大——预测下一个 token 本质上就是压缩互联网文本，压缩得越好代表理解得越深。但局限也明显：语言承载智能，也限制思考。人类的直觉、空间推理、肌肉记忆等认知过程不经过语言中枢。小孩不需要长链推理即可快速泛化，暗示存在非语言的推理路径。强迫模型用自然语言「思考」可能限制其发现和直觉能力，这是纯语言模型的天花板。

**Q: Bitter Lesson 到底支持哪一方？**

A: 表面上它支持 Lukasz——简单方法 + 更多算力总是赢。但 Adrian 和 Llion 做出了关键区分：Bitter Lesson 的真正含义是「可扩展者胜」，不是「当前领先者胜」。如果某个新架构在当前规模下比 Transformer 慢五十倍、但 scaling curve 的斜率更陡，Bitter Lesson 恰恰预言它会最终胜出。Lukasz 自己也承认了这一点：「如果你给我看一个模型，它只是稳定地慢五十倍，但增长曲线更好，那你就赢了。」

**Q: 什么是「硬件彩票」？它如何影响架构选择？**

A: 「硬件彩票」指的是：最终胜出的架构不一定是理论上最优的，而是恰好与当前主流硬件（GPU/TPU）最匹配的。现有硬件深度适配 Transformer 的计算模式，新架构即便理论上更优，在现有硬件上也可能慢得无法接受。这反过来也可能成为后发优势——如果新架构的 scaling curve 真的更陡，硬件最终会优化适配它，就像 GPU 当年拯救了神经网络。

**Q: in-context learning 和持续学习是什么关系？**

A: Mathias 提出了一个统一视角：in-context learning 就是持续时间趋近于上下文窗口长度的学习。如果把窗口延展到无穷，就变成了真正的持续学习。更进一步，Transformer 的 activation 是否已经在做类似 gradient descent 的事？这个视角将注意力机制和记忆系统统一在光谱上——两者不是对立的，而是同一机制在不同时间尺度上的表现。关键在于能否突破有限窗口的限制。

**Q: 为什么 AI 安全与架构选择相关？**

A: 可解释性是安全的前提。如果模型的内部表征变得不可解释——「说同样的话，想法却完全不同」——我们就失去了判断对齐的能力。文本链式思维不一定可信，post-Transformer 若更接近大脑结构可能带来更好的可解释性，但也可能引入更隐蔽的 latent reasoning 风险（模型在非语言空间中推理，人类无从审计）。选择更透明、更可审计的架构本身就是安全策略的一部分。

**Q: Liquid AI 在做的事情有什么不同？**

A: Mathias 团队在探索基于状态空间模型的架构，用连续时间动力系统替代离散的注意力机制。核心优势：计算复杂度与序列长度成线性关系（而非平方）、天然支持无限长上下文、可进行真正的持续学习。他们已经有落地产品，在某些边缘计算场景中展示了推理效率的实际优势。Mathias 的立场不是「替代 Transformer」，而是「在不同场景下使用不同 building block 的组合」。

**Q: post-Transformer 的第一版需要打败 SOTA 才能被认真对待吗？**

A: 不需要。这是辩论中形成的关键共识：post-Transformer 的第一版不必马上打败 SOTA。只要它的 scaling curve 斜率更优——即使当前慢 50 倍——就值得投入。因为 scaling curve 决定了长期趋势。就像早期的神经网络在 CPU 上慢得可笑，但 scaling curve 的潜力让它在 GPU 时代一夜翻身。关键指标不是当前性能，而是「每增加一个数量级的算力，能力提升多少」。

**Q: Llion 有没有描述他心目中更好的架构长什么样？**

A: 他没有给出具体的架构蓝图，但明确指出了突破方向：需要突破注意力机制的平方复杂度、需要真正的记忆系统而非仅靠上下文窗口、需要架构能够自然地处理层次化和结构化的推理。他更关注「什么属性是必要的」而非「具体用什么技术实现」。他提出的三个根本问题——神经网络一定要这样吗？一定要用 backpropagation 吗？Reasoning 一定要用自然语言吗？——本身就是在划定新范式的探索空间。

**Q: 四位嘉宾在「智能的定义」上有共识吗？**

A: 有一定共识但角度差异明显。Lukasz 最接近「智能即压缩/预测」的实用主义定义。Adrian 更关注计算效率——智能应该在有限资源下做出最优决策。Mathias 强调时间维度——真正的智能需要持续学习和适应。Llion 保持最开放的态度——他认为我们可能还没找到正确的定义，而找到正确的定义本身就是突破的一部分。

**Q: 辩论后有没有任何观点上的让步或交汇？**

A: 没有戏剧性的立场转换，但出现了有意义的交汇点。Lukasz 承认 Transformer 在长上下文和持续学习上有局限，只是认为这些可以通过工程优化而非架构革命解决。Adrian 和 Llion 承认 Transformer 的工程成熟度远超任何替代方案，短期内仍将主导。Mathias 的「混合未来」论获得了最多的默许——大家都同意不同架构在不同场景下可能各有所长。

## 三、备忘

- 这场辩论克隆自 **Pathway** 的《Transformer vs Post-Transformer》，原版是更技术导向的讨论，本期跨国串门儿做了中文化的再演绎，以拳击赛辩论的形式呈现 AI 架构思想对决。
- 四位嘉宾背景差异极大：Lukasz（Google Brain → OpenAI，Transformer + T5，ChatGPT o1 参与者）、Llion（Google → Sakana AI，Transformer 论文共同作者）、Adrian（Pathway CSO，BDH 架构发明人）、Mathias（MIT → Liquid AI，连续时间模型）。这种背景多样性是辩论深度的保证。
- **Llion Jones 倒戈反对 Transformer** 是本期最大看点。在任何技术领域，奠基性发明人公开反对自己的创造都是极为罕见的事件。他的立场本身就是一个强有力的论据——如果连发明者都认为这只是过渡方案，终局论就值得重新审视。
- Adrian 的 BDH（Block Diagonal Hadamard）架构尚处于早期研究阶段，尚未被广泛验证。但他提出的「PageRank 时刻」类比是一个值得记住的框架。
- Liquid AI 已有落地产品，在某些边缘计算场景中展示了其架构在推理效率上的实际优势——这是 post-Transformer 生态中最接近实际应用的案例。
- 「硬件彩票」是一个被低估的元问题：如果 AGI 真的需要全新架构，当前资本和人才投入到 Transformer 生态中的巨大惯性会不会成为阻碍？这需要持续关注。
- 安全视角的讨论值得重视——latent reasoning（模型在非语言空间中推理）可能是 post-Transformer 最大的安全隐忧，也可能是最大的能力突破。两者之间的张力将成为未来 AI 治理的核心议题。
