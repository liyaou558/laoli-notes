## 一、核心观点

### 1. Strong Opinion, Weakly Held：被「打脸」是早期投资的常态

戴雨森在上一集（#124）预言 2026 年是「Year of R」（回报、研究、记忆），并在年前清空了二级市场仓位。过去半年，他的部分观点确实被市场「打脸」了——尤其是 Anthropic 的 Coding 收入暴涨远超预期。但他认为，早期投资中被打脸是常态，行业变化越快，意味着机会越多。他引用二级市场的名言 **「Strong Opinion, Weakly Held」**：你可以有强烈的观点，但当支撑观点的前提发生变化时，真正的聪明人会调整自己的判断。他坦言，公开表达观点会增加认知调整的心理成本——「当你越强烈地表达一个观点的时候，你会被你说出去的话限制」——但这也正是他坚持录制这档系列节目的原因：把播客当作获得高质量反馈信号的「思想健身房」，逼迫自己持续学习、持续进化。

### 2. AI 叙事在 6-8 个月内已轮动了四五次，模型的「王座」从未坐稳

从 2025 年 11 月至今约 6-8 个月的时间里，AI 行业的主流叙事已经发生了多次剧烈反转：11 月 OpenAI 仍是王者（DAU 8 亿），12 月 Gemini 发布后 Google 被认为拥有最强算力和原生多模态优势，2026 年 1-3 月 Claude 的 Coding 能力异军突起、Anthropic 在二级市场估值一度超过 OpenAI，3 月叙事变成「Anthropic 一骑绝尘」，而到了 5 月 Codex 新用户增速超过 Claude Code、GPT 5.5 发布让 OpenAI 又卷土重来。戴雨森感叹，**「哪个模型更好」这个看似简单的问题，在不同月份问都会得到不同答案**。这既是 AI 行业的魅力，也说明头部模型公司之间始终没有拉开真正的决定性差距——谁后发模型，谁就占据后发优势。

### 3. 投入-产出-结果的三步框架：Return 问题并未真正解决

戴雨森构建了一个分析 AI 回报的核心框架：**投入（买 Token）→ 产出（做出软件）→ 结果（降本或增效，利润提升）**。Anthropic 年底预计 1000 亿美元 ARR、每月卖出约 100 亿美元 Token，在过去只是把它客户的「投入」当成了自己的「回报」。问题被推到了下游：烧了 Token 的人最后能不能赚到钱？他认为增效（增加收入）是一个渐进过程，不是程序员多了 10 倍就能立刻发生——「你突然有了很多便宜的编程能力，但你又不知道该做什么新东西的时候，第一反应是把别人做一遍」。降本（裁员）也只能做一次，不能持续产生增量。他用**水槽比喻**：涌入使用 AI Coding 的人远多于发现 Token 不值而离开的人，但迟早要赌住漏水口。硅谷大厂已经在举债建设数据中心，对回报问题的解答时间窗口是在变短的。

### 4. AI 硬件 7000 亿美元 Capex 还在加速，存储厂商利润已逼近英伟达

戴雨森披露了一组关键数据：2026 年 AI 硬件端的 Capex 已达 **7000 亿美元**。三星、海力士等存储厂商的利润已经接近英伟达，工人年终奖可达数百万。传统 CSP（云厂商）的利润已被存储厂商反超。Hyperscaler 们正在举债建设数据中心，使用资金杠杆。这意味着市场对 AI 回报的验证时间窗口正在缩短，整个链条的可行性正在被巨大的资本投入剧烈压缩——「这个数太大了，too big to ignore」。

### 5. Harness（编排层）正在变成 AI 的「操作系统」

戴雨森提出了一个贯穿整期播客的关键比喻：**模型是处理器（CPU），Harness 是操作系统（OS）**。就像 Windows 出现后，应用开发者不再需要直接处理 CPU 和内存的通信，今天的 Harness（如 Cloud Code、OpenCoder、Manus）封装了 agentic loop、memory、sandbox 等底层能力，让开发者只需在 Harness 上构建 Skill 或应用。用户可以像插拔 CPU 一样在 Harness 中更换模型——从 Opus 换成 Kimi K2.5 来降本，但对 Harness 中的记忆（memory）和配置更有粘性。一个关键信号是：已经出现了**基于 Cloud Code 做应用、而非直接使用 Cloud Code** 的趋势，这是操作系统生态诞生的雏形。他认为 Harness 变得更有价值这个事实，与「模型即产品」的论断并不矛盾——模型好才能做好 Harness，但没有好 Harness 就收集不到高质量数据来训练更好的模型，这是双向正反馈。

### 6. AI 投资：做 Horizontal、做创新、不打信息流战争

戴雨森给 AI 创业者两条核心建议。第一，**不要为了稳妥去跟风**——在变化极快的时代，跟风是最大的风险。第二，**要敢于做 Horizontal 而非 Vertical**——很多人受硅谷「垂直 SaaS」叙事影响，但那是 SaaS 后期通用机会被做完后的策略。技术发展早期，Horizontal 能力可以发散到多个场景（Lovable 从做网站扩展到做 PPT，Manus 从通用 Agent 衍生出网站/PPT/数据分析等场景）。他特别警惕「AI 信息流」产品——把 AI 生成内容套在字节跳动式的信息流壳里，等于在字节的游戏规则里跟字节竞争。**生产力方向（Agent 替代手工操作 Excel 等）是 10 倍-100 倍的提升，是简单题；2C 娱乐方向一开始就要跟抖音、小红书比谁更好看，是难题。**

### 7. 代理人网络效应：Agent-to-Agent Marketplace 可能是下一个大机会

戴雨森提出了一个前瞻性构想：**当每个人的 Agent 因积累了不同的 Context（专有知识、个性化配置）而产生差异化能力时，会出现 Agent 之间的雇佣市场**。同一任务给不同人的 Agent 做，结果不同。比如张小珺的 Agent 积累了她所有访谈的经验，戴雨森如果要准备采访，就可以付费雇佣她的 Agent 来生成问题——1000 元是 Token 成本，9000 元是专有知识的溢价。这个网络效应在六个月前还不成立，因为所有人的 Agent 跑的都是同一个 Opus，没有差异化。但今天 Harness 层的 context 沉淀使得 Agent 差异化成为可能。他还观察到已有公司在做 Agent-to-Agent Marketplace 的雏形。

### 8. 下一个字节跳动级的公司，长得不像字节跳动

戴雨森认为，移动互联网的几乎全部「金科玉律」在 AI 时代都可能被颠覆。**核心公式从「Attention is all you need」变为「Attention is not all you need」**——移动互联网的商业模式是 DAU × 时长 × 变现效率，但 Agent 时代的核心指标变成了「Agent 能够持续完成有价值任务的长度」。封闭生态的护城河可能变成软肋（微信里的信息 Agent 看不到，用户就有动力迁移到飞书）。广告模式面临根本性质疑：如果内容都是 AI 看的而不是人看的，广告还存在吗？他强调，把上一个时代的成功范式（信息流、投放获客、DAU 考核）搬到 AI 时代，就是在和上一个时代的冠军在它的游戏规则里竞争。

## 二、关键问答

**Q: 你去年底的「Year of R」判断，哪些看对了，哪些看错了？**

A: 看对的部分主要是 C 端用户的担忧——ChatGPT 的订阅付费用户增长已进入停滞（用户从 20 美元提到几十/一百美元很难，5000 万付费用户已经把愿意为 AI Chatbot 付费的人群筛选完了），广告和电商的推进也远低于预期（Google 和 Facebook 都花了 6-8 年才成为广告巨头）。看错的部分是完全没预见到 Anthropic 的 Coding 收入会在短时间内爆发。戴雨森用「烧开水」比喻来解释：AI 能力是智力，突破某个阈值才会突然产生巨大价值，就像水烧到 100 度才会有蒸汽机，但到底多少度是烧开，提前预判很难。

**Q: 为什么 Anthropic 能在 Coding 上实现突破？除了技术还有组织原因吗？**

A: 戴雨森的观察是：AI 行业从探索期进入了主线收敛期。探索期适合自下往上、发散型的组织（如 OpenAI 的多个方向并发、Sam Altman 像天使投资人一样鼓励各种小项目），而高速赛跑期需要从上往下、价值观一致、方向齐心的组织。Anthropic 有价值观面试，CEO Dario 每两周发内部 MEMO 与全公司同步思考，是「管杀也管埋」的组织；OpenAI 很多产品是「管杀不管埋」——GPTs、Sora、Operator 等上线后缺乏持续维护。但如果一两年后新范式诞生，可能又会回到需要发散型组织的阶段。

**Q: Anthropic 的优势是稳固的吗？它和 OpenAI 谁被高估谁被低估？**

A: 这需要分短期/中期/长期来看。短期（三个月）：两家可能都被低估了——如果现在上市，市场普遍认为至少值 2 万亿美元，而 Anthropic 二级市场新股价格约 9000 亿美元，OpenAI 也差不多，年底 ARR 预期 1000 亿美元才 10 倍，上市可能直接翻倍到 3 万亿。中期（一两年）：可能被高估，因为 Return 问题并未真正解决，只是换了形式。长期（十年）：赢家可能有 5 万亿-10 万亿美元。

**Q: 你怎么看 Claude Code 和 Codex 之间的竞争？**

A: 目前两者在做很多任务时没有本质差别。品牌锁定和用户粘性很重要——Cloud Code 用户配置了大量 Skill 和 Claude.md，迁移到 Codex 需要重建。Codex 采取了价格战策略，同等体量下比 Cloud Code 便宜约 50%。两个月前 Codex 明显落后，但 GPT 5.5 改变了局面。现在竞争变成了「渠道之争」——品牌、渠道、价格变得更重要。好马配好鞍，模型和 Harness 通过 Coding Plan 绑定在一起。

**Q: Agentic Coding 提升效率 10 倍，但为什么公司收入没有相应大涨？**

A: 戴雨森的核心论点：大多数公司的问题不是缺程序员写代码，而是不知道该做什么新产品。对于有规模的公司，突然多了 10 倍工程师，收入并不会大涨——因为新产品的出现、新需求的发现是一个渐进过程，不是编程问题而是创新问题。他观察到硅谷公司开始互相进入彼此的领地（Lovable 做 PPT、Gamma 做网站），本质上是有了编程能力后第一反应是「把别人做一遍」，而不是想出新东西。

**Q: Harness（编排层）到底是什么？为什么突然变得重要了？**

A: Harness 的本意是马具，引申为「让一个非常强的东西在你规定的范围内运行」。戴雨森将 Harness 分为几个层次：最底层是模型本身；外面是 Context 层（实时的、组织专属的信息）；再外面是工具和 Agentic Loop（模型的运行循环）；最外面是 Sandbox（运行环境）。OpenCoder 就是通过给模型更多权限（访问 Mac 文件、日历、信息）、心跳机制（每 30 分钟检查任务）、Memory MD（长期记忆）等工程创新，解锁了模型公司没想到的场景。过去大家叫「套壳」，现在发现这些工程层的创新非常有价值。

**Q: 现在大家都在做 Agentic Coding，是否过于同质化了？**

A: 戴雨森认为确实有同质化风险，但他有一个「暴论」：AGI 的定义在不断缩水——从毁灭人类的 Singularity，到解决黎曼猜想，再到现在「替代普通程序员」。写代码尤其是日常编程，本质上是 within distribution 的任务。真正 OOD（out of distribution）的创新——新产品、新药物、新发现——仍然是开放问题。有些公司做 Coding 是「蒸馏」已有数据（抄答案），而不是做创新的过程。现在 AI 毁灭旧价值的速度远高于创造新价值的速度。

**Q: 2026 年创业是更难了还是更容易了？应用创业者的机会缩小了吗？**

A: 从资本市场看，投的应用公司数量在变多、估值在变高（很多公司一出来就几亿美金）。从原理看，模型能力和 Harness 基础设施都在变强，应用可做的场景变多了。真正的问题是：大厂（包括模型厂和互联网巨头）做应用创新的速度其实很慢，历史上伟大的创业公司都有某种「大厂看不上」的属性——太小众（Airbnb、B站）、太 low（拼多多、内涵段子）、太累（美团）、太超前（OpenAI、SpaceX）。Harness 去年被普遍认为是无价值的「套壳」，这个被低估的领域正好给了创业公司「认知差」的机会。

**Q: 你们怎么投人？有什么不投的清单？**

A: 真格一直以投人为核心。戴雨森分享了内部的两套框架：创始人四分类（小天才、老司机、科学家、操盘手）和四力模型（学习力、创造力、领导力、意志力）。最看重的是：对技术有前瞻的原创判断（不是听播客听来的）；具备企业家精神且敢于放弃安稳出来创业。不投的是跟风项目——看到赛道火了、VC 在谈、播客在讲，就做一个「better」版本的产品。他们去年看了约 100 个项目投了 2 个，今年到 5 月已投了 3 个，节奏比去年稍快。

**Q: 硅谷 VC 投资最火的主题是什么？中国有什么不同？**

A: 硅谷三个最火的方向：①Coding 带来的大跃进式烧 Token（Meta 有 Leaderboard 比谁烧得多）；②世界模型（机器人领域，大家想把语言模型的范式——Pre-Training / Fine-Tuning / RL——平移到机器人模型上）；③Auto Research（AI 自我迭代、AI 提升 AI）。硅谷有约 60 多家 New Lab。中国方面，机器人、世界模型、AI 硬件（可穿戴等）都很火，硬科技有资本市场的正反馈。但戴雨森观察到硅谷非常聚焦在模型和 Vertical SaaS，Horizontal 的通用应用相对空白——Manus 在硅谷几乎没有竞品。

**Q: 你对消费级 AI 硬件怎么看？投了吗？**

A: 基本没有投。戴雨森有「暴论」：很多 AI 消费电子会重蹈新消费覆辙——给硬件加个 Chatbot 或 AI 感知能力，但没有创造新的需求品类。做硬件比做软件难得多（供应链、开模、设计、资金占用、销售）。他认为长期来看，AI Agent 最好的载体仍然是手机和云端，因为现有设备在尺寸、性能、耗电、散热上已做得很均衡。可穿戴设备需要用户养成坚持佩戴的习惯，目前增量价值（录音、提醒 todo）相对于麻烦来说很有限，「很多时候是个 VC 想象出来的需求」。

**Q: 「蒸汽机到电动机」的类比说明了什么？**

A: 蒸汽机时代的工厂围绕一根主轴建设，呈狭长型。电动机出现后，电线替代了主轴，厂房可以建得更大更扁平——这才催生了福特的流水线，生产效率大幅提高。戴雨森用这个历史类比说明：AI 技术要真正释放生产力提升，需要组织的物理变化（部门墙的拆除、分工的重组），而组织变化是人的变化，「不是一分钟就变了，可能要十年才会变，甚至要新的组织替代老的组织」。这就是为什么大公司推 AI 不是简单装个 Cloud Code 就完事了。

**Q: AI 时代的教育和人才培养方向是什么？**

A: 戴雨森认为 Taste（审美/品味）可能也不是人类的专属阵地了——AI 见过的优质数据远超普通人，做的 PPT 比人好看。核心能力回归到 **Agency（主观能动性）**——不管 AI 做得多好，「要做什么」还是人来决定。他观察到很多人用 OpenCoder 有一种「养成系」的情感投射，人和 AI 从「聊天」变成了「合作」，关联性和依赖度都大幅增强。对于孩子教育，他坦白没有答案，但认为培养好奇心、创新性、以及「问出好问题」的能力至关重要。他提出了一个深刻洞察：AI 基于现有模型结构能很好解决 within distribution 的事情，但创造 out of distribution 的内容（如原创笑话、伽罗华 20 多岁搞群论）仍然是人类的独特能力。

**Q: 你还会继续录这个系列吗？怕不怕再被打脸？**

A: 戴雨森引述即刻瓦总（西东）说服他的话：「如果你认为观点被打脸就不愿意再说了，那你是真正被你的观点绑定了。你应该是一个持续学习、持续进化的个体。」他把录制播客当作「思想的健身房」——刻意练习思考、通过表达发现自己哪些地方没想清楚。他说：「作为一个早期投资人，**平凡被打脸是幸福的**。如果一个人总是不被打脸，大概不是因为他的观点正确，而是因为他没有进步。」他会继续这个系列。
