---

## 一、核心观点

1. **"融到钱就是创始人的愚昧之巅"——刚拿到融资时的自信往往是幻觉。** 梦琪坦诚复盘，去年刚拿到融资时，她觉得自己"英明神武"，逻辑上无比自洽——选垂直 Agent、讲 Io 算法故事、招顶尖科学家、造各种新概念。但做了六个月后发现，当初那些"特别对"的决策几乎全是错的。她将那段状态称为"愚昧之巅"——"觉得自己讲得太好了，我怎么这么聪明，我怎么这么能造概念"。真正的创业不是靠逻辑推演能赢的，落地时会发现现实"非常恶心"。

2. **"虚荣心是创业最大的绊脚石"——迎合市场和投资人口味的决策必然失败。** 梦琪承认自己初期大量决策"不是基于真正的市场需求"，而是基于"我得装一下，得满足自己的虚荣心"。她招了算法科学家，但作为纯应用公司根本用不上——"你让人家给你在这干什么呀，神经病就纯属，给投资人就是说我家这么一个人"。她反思创业者的牵引力来源应该只有用户，而不是投资人、同行、或者市场热点。把创业做成"证明题"会无比容易——目标函数清晰、约束条件基本没有——但做出真正好的产品极难。

3. **"垂直 Agent 面临结构性的困局，toB 不值得创业。"** 梦琪团队做了四五个月的达人营销 Agent 后，发现两个系统性困局：第一，垂直 Agent 公司被迫成为 agency——客户不用产品，但你想赚钱，只能出人去服务，产品沦为给投资人看的"景观"；第二，toB 的组织形态和年轻创意团队格格不入，"被包养就不要谈独立人格"。她甚至去美国调研了一个多月，得出的痛苦结论是：华人在美国做 2B 天花板非常明显。最终她认定 toB "不值得创业"，因为她不想"站在半空中看棋盘选机会"，而要的是"老娘今儿就干这事，不干我难受死"的状态。

4. **"软件是一块需要持续修剪的草坪"——模型变强不等于产品好做。** 梦琪引用美国创业者 Brad Taylor 的比喻：软件像草坪，修剪它的成本比放在那里更高。她自己的 Clico 客户端产品做了三个月、49 个版本才勉强"能用"。她反驳"几周就能搓一个同类产品"的论调：过去一年做过 PR 的软件公司中，产品能用、不挂、顺畅的不到 10%。"软件是由 15000 个细节组成的，你就是一点点去把那 15000 个屎吃掉，多吃一个，用户体验好一点。"

5. **"只要你能缩短用户意图和解决方案之间的距离，你就有价值。"** 梦琪起初看不起自己的浏览器插件产品 Clico，觉得就几个按钮，"这有什么意义"。但她后来发现，Clico 把用户在多个窗口间截屏、写 prompt、复制粘贴的流程，压缩成一个按钮——按下去就代表意图。用户不需要说任何话，"你按完之后代表他不用写那个 prompt"。她总结 C 端产品的核心就两个字：体验。70% 的精力应该花在交互上，既要更快更便捷，也要提供情绪价值。

6. **"AI 渗透率还极低，世界非常参差，这是乐观的理由。"** 梦琪在旧金山做田野调查，逢人就问是否知道 Whisper Flow——在 101 公路上打了巨大广告牌的产品——结果只有 40% 的人知道，其中仅 20% 的人用过。她发现 Reddit 上每天都在吐槽 context switching 的痛苦，但 AI 连这种"简单问题"都没解决完。她同时观察到，大量普通企业连 workflow 都没用上，更别说 agent。"这个世界在一个动态的博弈和竞争里面运转，不是一篇文章说谁把谁干死了就真的干死了。" 她称之为"AI 行业的红果短剧"——自媒体把漫长枯燥的竞争过程压缩成几秒钟的戏剧冲突。

7. **"用心去做事，而不是用脑子——左脑不能用太多。"** 梦琪创业近一年最大的体会是：创业没有科学的变量和因子，融钱多、团队豪华、赛道热门都不是充分必要条件。她经历三四月份最痛苦的"虚无期"后，在 Clico 这个产品上找回了热情——"这个产品在垃圾，你不管你怎么骂我，我就爱这个产品"。她不再想做效率工具，因为效率工具"只关注你的问题，不管你是谁"；她想做一款"首先能解决问题，同时在乎你是谁、你现在心情怎么样"的产品，把用户当人而不是 DAU 数字。

8. **"用 Reddit 做用户研究，能让创业者不焦虑。"** 梦琪分享了她的用研方法论：每一个产品决策拆成决策漏斗，每一层在 Reddit 发 40-50 个帖子，回收所有反馈。不提产品，单纯讨论需求和痛点。这种方法让她不再"放肆地塞流行概念"，比如 Chat UI 和 GUI 怎么结合、数字员工等——"用户说你吃饱撑的，先解决我问题好不好"。她建议所有创业者都去泡 Reddit，因为"你会发现很多问题没有被解决，AI 这么牛逼，怎么还没解决这些这么简单的问题"。

9. **"如果不做世界首发的 PR，一定是因为你对产品不满意。"** 梦琪团队没有做大规模宣发，表面原因是"比较抠"，但深夜自问时她承认：对那个 2B 产品不满意、不相信，所以没法去大肆宣传。而现在对 Clico，她有了真正的 Spark——"这就是人内心深处的那种火、那种光，AI 解决不了"。她认为过程产品反而更能代表公司的精神气质，也让自己有机会"把个人的世界观、价值观、性格投射到一个产品里面"。

## 二、关键问答

**Q：你创业也快一年了，之前做过 2B 的 SaaS，也做过 2C 的事，去年创业的状态是什么样的？**
A：（梦琪）我去年大概这个时候开始创业，创业之前在 SaaS 最火的时候做过几年 2B 解决方案，后来在 TikTok 做 2C，甚至还做过一年 HR。我一直抱着"能多学点多学点、学完自己好用"的心态。但这一年走完了很多软件 AI 创业者的典型流程——融钱、组团队、讲故事、然后开始 pivot。到今天回头看，软件行业确实到了一个低点。（曲凯补充）23 年整个行业都低，24 年巨身热但软件低，25 年巨身热到大家质疑泡沫，今年巨身比去年还热但软件不如去年热。这就是当下的背景。

**Q：为什么一开始选了垂直 Agent 而不是 General Agent？**
A：（梦琪）我们成立前就决定做 Vertical Agent，原因很简单——觉得 General 太拥挤，那会儿已经有"第 18 个 Manus 出现了"，我们不想加入混战。当时觉得这个逻辑怎么说怎么对，"我太英明神武了、太聪明了"。但后面发现那些所谓的"特别对的决策"都是错的。第一个错就是选 Vertical，第二个错是选达人营销作为切入点。

**Q：做达人营销 Agent 时遇到了什么困境？**
A：（梦琪）做了一两个月后发现产品长得就是 SARS——跟八年前开始创业的 SaaS 公司产品一模一样。我们自己没有高质量数据，用人家的 API，做了一个差不多的界面，"纯属脑子有病，等于把人家 SaaS 走过的路又走了一遍"。两个核心问题：一是如果服务专业用户，他会一直提需求直到产品跟 Adobe 差不多，必然走向 SaaS；二是 sourcing 只占用户工作的 30%，你在巨大价值链上选了一个 AI 的 low-hanging fruit，后面最浪费时间的来回去聊的过程被 skip 掉了。到 12 月份我清晰地感受到——Trade Agent 不成立。

**Q：为什么不往 Pro C（专业消费者）方向走？**
A：（梦琪）大家常把 Pro C 和 OPC（独立开发者）混在一起。OPC 是趋势但不是现实——现实中的独立开发者首先很少，挣到钱的更少。没挣到钱凭什么给你钱？做独立开发者的生意其实挺艰难。而且最有信息 access 能力的 Pro C，可能把 Cursor 用得特别好，留给我们的机会没那么多。C 端生意分两种：一是 Token 生意（做大 R 运营，让月付 5 万美金以上的人留在平台），二是健身房生意（赌用户订阅了不来、Token 不打满）。我们想做健身房模式，不做没毛利的业务。

**Q：去美国调研后有什么发现？**
A：（梦琪）我彻底对美国创业生态"去魅"了。那边跑流水的情况比我想象中严重多了——六个人互相买、非常复杂的机制，你审计不出来。每个月底都有人疯狂找我吃饭，原来是想充流水。如果我参与进去，PPT 上就能挂好几个美国头部 SaaS 公司的客户 logo。但我没参与，觉得不太 mixed，要维持自己的道德标准。这让我更清楚：如果创业是服务虚荣心，目标函数清晰、约束条件基本没有，事情会变得无比好实现。

**Q：从 2B 转 2C 的具体过程是怎样的？**
A：（梦琪）2 月份我们低功耗经营 2B 产品，只留了一点维护的人力。但在此之前我们已经开始试水 C 的事——因为"团队的士气和信心是不可再生之物"，我不能突然说咱不干了。我们先做了一个特别简单的浏览器插件叫 Clico，只解决一个问题：你浏览器开一堆 Tab 切来切去找不到信息。后面加了 writing 能力，能跟随光标随时唤起。就这么一个我能说出 100 个替代品的产品，仍然有用户喜欢——南非的用户说"我终于能沉浸式干一件事了"。

**Q：为什么说"软件是草坪"？**
A：（梦琪）这是美国做客服 Agent 的公司创始人 Brad Taylor 说的。软件像草坪，修剪它的成本比把它放在那儿更高。我自己用 bolt.new 搓一些小工具自用，发现做完以后要维护它，总能遇到 corner case——创造的快乐很快就结束了，剩下的都是痛苦。我们也试过一个多 Agent 协作的产品但没发布，因为我不想做平台型产品等着用户来演绎，我不是擅长做社区的人。

**Q：你们怎么做用户研究的？**
A：（梦琪）最大程度用好 Reddit。一方面做数据挖掘拿静态结论，另一方面每一个产品决策拆成决策漏斗，每一层发 40-50 个帖子，回收反馈。不提产品，因为提产品的流量都不好，单纯讨论需求和痛点。比如 context switching 的问题，Reddit 上每天有人在吐槽——"我每天在这几个 context 里切换，真的要人格分裂了"。泡 Reddit 最大的价值是让你不焦虑，因为你会发现 AI 这么牛逼，连这种简单问题都没解决完。

**Q：你做产品时最大的教训是什么？**
A：（梦琪）克制住"一定要装逼和保持领先"的心态。我们做 2B 产品时追了很多流行概念——Chat UI 和 GUI 怎么结合、数字员工、Multi Agent 架构——觉得"我太高明了，没有人比我更懂 AI"。但用户说"你吃饱撑的，先解决我问题好不好"。现在我花 70% 的时间思考需求到底是什么、怎么最好地解决它，然后才想能不能把新的 AI 概念塞进来。"第一性原理是解决问题的最短路径，其他的都是周边。"

**Q：软件创业者当前最大的焦虑是什么？**
A：（梦琪）最大的牵引力就是 AI 模型的发展。模型一变强，大家都会被迫 pivot。我们做客户端产品适配各种版本和型号，光打磨就需要三到六个月，但产品生命周期可能只有一年多。这完全改变了思维方式——"你别提明年了，就下个月吧"。我因为这个产品被气哭过好几次，在 Palo Alto 的 Blue Bottle 调产品没调出来气得锤桌子，结果隔壁桌站起来说"能让创始人这么 emotional 的一定是好玩的东西"。

**Q：你觉得 AI 软件创业还有机会吗？**
A：（梦琪）非常乐观。第一，渗透率极低——我在旧金山逢人就问 Whisper Flow，仅有 40% 的人知道，用的人更少。路边随机问用不用 ChatGPT，比例也不高。第二，世界非常参差——大部分企业连 workflow 都用不上，别说 agent 了。第三，竞争不是一瞬间完成的——抖音出来快手不会立刻死，商业社会是动态博弈。虽然行业结构决定大部分利润被头三家吃走，但"因为这个趋势动量太明确了、渗透率太低了，你可能喝的这个汤比你过去做 SaaS 吃的肉还要多"。

**Q：为什么没有做大规模宣发？**
A：（梦琪）三个原因。一是我去年比较低调。二是 Manus 之外，做世界首发的产品大多一波流，留存不好，有点浪费钱和交作业的意味。三是最深层的原因——我对那个产品不满意、不相信。现在做 Clico，"这个产品在垃圾，你不管你怎么骂我，我就爱这个产品"。深夜闭上眼睛问自己——这就是我的 Spark，是 AI 解决不了的人内心深处的那团火、那道光。

**Q：如果再选一次，还会创业吗？**
A：（梦琪）官方版本：会创业，我很 enjoy，也很擅长，meant to be。非官方版本：三四月份我最痛苦的时候，无数次问过自己，为什么创业？不创业在字节的工作也挺好的。那段时间看到行业陷入无尽虚无，真想掀桌子算了。但最近热情慢慢回来了——"我要是做不出来，别人也做不出来。但如果有一个人做出来，那一定是我。" 创业就是这样一个 up and down，底色是热爱。

**Q：融资多、团队豪华，这些是创业成功的充分条件吗？**
A：（梦琪）都不是。新消费那一波融得最好的可能现在最惨，因为回归也回归不掉、收入扛不住估值。有大牛的公司，要么挂名、要么落地不够好，或者赛道不对。我越创业越觉得这事挺玄学的——你找不到一个科学方法、一个变量公式。"用心去做这个事情，而不是用脑子。左脑不能用太多。把你的用户当人。" DAU 对我来说不是一个四位数，背后都是人。你跟他聊得足够多，他在你心里就更立体，你会更体恤他。

**Q：你现在对 AI 行业的自媒体内容怎么看？**
A：（梦琪）我特别讨厌一类文章，就是"AI 把某某干死了"那种。商业社会里的竞争不是一瞬间完成的，不是抖音出来快手今天立刻死。现在的自媒体喜欢把漫长枯燥的竞争过程压缩成几秒钟的短剧剧情——我称之为"AI 行业的番茄、红果短剧"。但真实的商业世界不是只有 early adopter 活着的地方。

## 三、备忘

- **梦琪的核心产品历程**：达人营销 Agent（Sourcing）→ 全链路增长 Agent → 低功耗维护 → Clico（浏览器插件，MVP 阶段跑出正反馈）→ 客户端产品（49 版迭代，持续打磨中）→ 内部达人营销 Skill（过程产品，反而比花五六个月做的正式产品效果更好）
- **关键人物/公司提及**：Manus（带动 Agent 叙事）、Mainless/Jandle（工程质量标杆）、Brad Taylor（"草坪"比喻来源）、Whisper Flow（旧金山田野调查样本）、Monica（Clico 的对标参考）
